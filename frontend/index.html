Core Capabilities

Built for Reliable Healthcare AI

Every feature designed to prioritize accuracy and patient safety over artificial intelligence creativity.

Long-term Memory

Patient data stored as semantic embeddings in Qdrant vector database. Memories persist and are instantly retrievable.

üìä

Evidence-based Reasoning

LLM answers grounded exclusively in retrieved memories. Every response is traceable to stored patient data.

Anti-hallucination

No evidence, no answer. The LLM is only called when relevant memories exist in the database.

Patient Isolation

All queries scoped to patient_id. Complete data isolation ensures patient privacy and prevents leakage.

Evolving Memory

Confidence reinforcement for frequently accessed memories and time-based decay for outdated information.

Quick Start

Try the API

Simple REST API with three powerful endpoints for memory management.

üìù 1. Store Memory 
curl -X POST http://localhost:8000/api/v1/memory \
  -H "Content-Type: application/json" \
  -d '{"patient_id":"patient_001",
       "raw_text":"Anxiety and trouble sleeping",
       "memory_type":"mental_health",
       "source":"session"}'
üîç 2. Search Memories 
curl -X POST http://localhost:8000/api/v1/search \
  -H "Content-Type: application/json" \
  -d '{"patient_id":"patient_001",
       "query":"anxiety symptoms"}'
üí¨ 3. Context Answer 
curl -X POST http://localhost:8000/api/v1/search/context \
  -H "Content-Type: application/json" \
  -d '{"patient_id":"patient_001",
       "query":"What symptoms were reported?"}'

# Returns grounded answer OR "Insufficient data"
